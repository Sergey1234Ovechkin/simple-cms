    Working with user events

    This article will cover the basic principles of working with user events in Htmlix and create a small application of four components: user name input forms, greetings, exit buttons, and an array with application users.

    A ready-made example can be called here

    After entering the user's name into the form, the application will display his name in the greeting, exit button and create a new container in the array with users, as well as save the data in localStorage. After reloading the page, it will take the name from localStorage. When you click on the "log out of profile" button, it will delete data from all components and a variable from localStorage.

    Let's create four components: an entry form - form, a greeting - greeting, an exit button - logout and an array with all visitors - users_array, and see how we would create communication between them if we did not use user events.

    Let's create html markup of all components:

    <!-- container component form with two properties input with the property type "inputvalue" and click with the type "click" -->

<form data-form="container" class="card col-12">
 <div class="form-group">
 <label for="">Enter a name</label>
 <textarea data-form-input="inputvalue" class="form-control" rows="1"></textarea>
 </div>
 <button data-form-click="click" type="submit">Submit</button>
</form>


<!-- container component greeting and user_name property with type - "text" -->

<div data-greeting="container" class="col-6 card"> 
 <p>Hello: <span data-greeting-user_name="text">guest</span></p>
</div>


<!-- container component exit button and user_name property with type - "text" -->

<div data-logout="container" class="col-6 card"> 
 <a href="#"> Log out of profile: ( <span data-logout-user_name="text"></span>)
</a> 
</div>


<!-- component is an array of users, initially with two containers, in each container the user_name property is "text" --> 

<div class="container-fluid" style="border: 1px solid red; margin-top: 20px;">
 <p> all users:</p>

<div data-users_array="array" class="row">

 <div data-user="container" class="col-4 card"> 
 <p>user - 
 <span data-user-user_name="text">user_name_1</span> 
 </p> 
 </div>

 <div data-user="container" class="col-4 card"> 
 <p>user - 
 <span data-user-user_name="text">user_name_1</span> 
 </p> 
 </div>

 </div> 
</div>

    In the example above, we wrote the data of the "input" property from the form component to the "text" variable. Now we need to display them in the properties of the greeting and logout components, as well as create a new container in the users_array array. To understand why custom events are needed, let's first try to do this in various ways without using them.

    1. method - in the click method of the form itself, go directly to each component and set the text property:

     event.preventDefault();
 var text = this.parent.props.input.getProp();
//set values to properties
 this.rootLink.state["greeting"].props.user_name.setProp(text);
 this.rootLink.state["logout"].props.user_name.setProp(text);

 ///created a new container
 this.rootLink.state["users_array"].add({user_name: text});

    The disadvantage of this approach is obvious, what if we then get data for this property somewhere else? For example, based on localstorage, they will be loaded automatically.

    2. the way is to create a method in the stateMethods object and call it when the data changes:

    stateMethods: {

 //this in stateMethods methods points to rootLink
entryUser: function(text){

 this.state["greeting"].props.user_name.setProp(text);
 this.state["logout"].props.user_name.setProp(text);
 this.state["users_array"].add({user_name: text});
 }
}

// after loading the page
window.onload = function(){
 var name = window.localStorage.getItem('user_name');

 if(name)HM.stateMethods.entryUser(name); ///calling the method
}

//in the form
of click: function(){
event.preventDefault();
 var text = this.parent.props.input.getProp();
 this.rootLink.stateMethods.entryUser(text); 
}

    In this case, we managed to avoid code duplication, but the problem is that the methods for working with the properties of the component are not inside it, but in external functions. What if, before setting a property, it needs to be formatted somehow, differently for each component, or we will need to set not one property, but several, so it's better when the components themselves work with their properties.

    3. the way is to transfer all the logic of working with properties to components, and create methods in them with which to call certain actions.

     greeting: {

 container: "greeting",

 //added an auxiliary method for greeting the user
 props: [ "user_name", ["greet_user", "aux"] ], 
 methods: {

 greet_user: function(name){
             this.props.user_name.setProp(name+" !!!");
       }
    }
 },
container: "logout",

 props: [ "user_name", ["set_name", "aux"]], //added an auxiliary method to work with
the methods component property: {

 set_name: function(name){
                      this.props.user_name.setProp(name);
                      }

           }, 
 },

users_array: { 
 arrayProps: [["entry_user", "aux"]], ///adding a container with a new user from the array
 arrayMethods: {
 entry_user: function(name){
          this.add({user_name: name})
        }
}
container: "user",
props: [ "user_name", ],
methods: {
         },
},
stateMethods: { //changed the general
entryUser: function(text) method{

 this.state["greeting"].methods.greet_user(text);
 this.state["logout"].methods.set_name(text);
 this.state["users_array"].methods.entry_user(text);
  }
}
//then we also call this entryUser method from the form and when loading the page

    Now the components are encapsulated, but we access the component by its name. What if we then want to change the name of the component or method, or delete the component altogether, or we will have many such components that listen to the same property. Or we will have many such properties, respectively, we will have to create many such methods.

    To facilitate this task, you can use custom events. The component subscribes to the event, and when it occurs, it does something with its properties.

    Let's edit the code, now using custom events:

    var StateMap = {

 eventEmiters: { 
 //created a user login event emitter 
 ["emiter-entry-user"]: {prop: ""}, 
 },
 form: {
 container: "form",
 props: ["input", "click"],
 methods: {

 click: function(){
 event.preventDefault();
 var text = this.parent.props.input.getProp();

 //called an event in the form and passed data to it
 this.rootLink.eventProps["emiter-entry-user"].setEventProp(text); 
 window.localStorage.setItem('user_name', text); 
 } 
 }, 
}, 
greeting: {

container: "greeting",
props: [ "user_name", ['listen_entry_user', "emiter-entry-user", "" ] ], //added an event listener "emiter-entry-user"
methods: {

listen_entry_user: function(){
//received data from the event and updated property
 this.parent.props.user_name.setProp( this.emiter.getEventProp() );
 }, 
 }, 
},
logout: { 

 container: "logout",
props: [ "user_name", ["listen_entry_user", "emiter-entry-user", "" ]], //event listener property "emiter-entry-user"
methods: {
listen_entry_user: function(){
//got data from the event and updated the property 
 this.parent.props.user_name.setProp( this.emiter.getEventProp() );
},
},
},
// here the event listener is added to the array property, because if you add it to the container, it will be called for each container 
users_array: { 

 arrayProps: [ ['listen_entry_user', "emiter-entry-user", ""] ], //event listener property "emiter-entry-user"
 arrayMethods: {

listen_entry_user: function(){
//got data from the event and created a new container
 this.parent.add( {user_name: this.emiter.getEventProp()} );
           }, 
 }, 
 container: "user",
 props: [ "user_name", ['listen_exit_user', "emiter-exit-user", ""] ],
 methods: { 
            }
 }, 
},

}
window.onload = function(){

var HM = new HTMLixState(StateMap);
 var name = window.localStorage.getItem('user_name');

 if(name != null)HM.eventProps["emiter-entry-user"].setEventProp(name); ///called the "emiter-entry-user" event when loading the site and passed data
to it }

    Thus, we got rid of the intermediate method "entryUser" and simply call the event by passing new data to it in the form and after loading the page.Now if we need to change something in a component, or delete it altogether, we do not need to correct the code in each function, which can be as many as you want. And to unsubscribe from any event, you can simply delete the listener in the component, or temporarily disable it using the disableEvent() method in the subscriber property.
