  "title": "Loading templates from .js file - templateVar",
  "text": "The first way to connect the missing templates is to use fetchComponents, with this method, first those components that are in the markup given by the server are created, and the rest asynchronously, after loading the templates.\r\n\r\nThe second way is to put the templates in .js file for example template.js, connect it to index.html and specify in the settings:\r\n\r\nfile - index.html :\r\n\r\n```\r\n <!--------------------->\r\n<script src=\"./template.js\"></script>\r\n\r\n```\r\nStateMap settings\r\n```\r\nstateSettings :{\r\ntemplateVar: template, ///variable name from the template file.js\r\n} \r\n\r\n```\r\n\r\nThe template file itself.js :\r\n\r\n```\r\n\r\nvar template = `\r\n<!--- variant_cont_2 container ------------>\r\n <div data-variant_cont_2=\"container\" style=\"border: 1px solid green;\">\r\n <p data-variant_cont_2-text=\"text\" data-variant_cont_2-style=\"style\" style=\"color: green;\" > text of option 2 </p>\r\n <p data-variant_cont_2-text2=\"text\" > additional text of variant 2 </p>\r\n</div>\r\n\r\n<!--- container variant_cont_3 from the virtual array variant_arr_3 ------------>\r\n <div data-variant_cont_3=\"container\" style=\"border: 1px solid green;\">\r\n <p data-variant_cont_3-text=\"text\" data-variant_cont_3-style=\"style\" style=\"color: green;\" > text of option 3 </p> \r\n </div>\r\n \r\n `;\r\n\r\n```\r\nNow all components will be created synchronously.\r\n\r\nIf the router is used in the application, then the first and templatePath fields can be omitted, because they will not be used, for example:\r\n\r\n```\r\nvar routes = {\r\n\r\n[\"/\"]: {\r\nroutComponent: {\r\n\r\nrouter_main: \"home\", //components corresponding to this route\r\n\r\n      },\r\n},\r\n\r\n[\"/\"+SITE_NAME+\"/\"]: {\r\nroutComponent: {\r\n\r\nrouter_main: \"home\", //components corresponding to this route\r\n\r\n        },\r\n}\r\n\r\n\r\n```"
