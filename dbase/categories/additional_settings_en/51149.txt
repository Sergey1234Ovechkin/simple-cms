  "text": "There are two ways to specify properties:\r\n\r\n* 1 - Specify the property in the html markup using `data-container_name-prop_name`, or `data-array_name-prop_name` - for the array properties. \r\nFurther in the description of the application, we specify it in the props array: `[\"prop_name\", ...]`, or `arrayProps: [\"prop_name\", ...]`- for array properties.\r\n\r\n* 2 - Specify the property only in the application description using the array `props:[[\"prop_name\", \"prop_type\", \"selector\"] ]`.\r\nwhere \"selector\" is a selector for searching for a property relative to a container, or an array for array properties, for example `\"a:first-of-type\"`. \r\nIf the selector is set to empty `\"\"`, it will mean that the property is in the same tag as the container, or an array is for array properties.\r\n\r\n\r\nFrom the example above (# Listening to user events from array properties), the properties can be specified as follows:\r\n\r\n* for array properties: `arrayProps:[ [\"class_menu\", \"class\", \"\"], [ \"listener_load_page\", \"emiter-load-page\", \"\" ] ]` - the same tag as the array;\r\n\r\n* for container properties: `props:[[\"class_item\", \"class\", \"\"], [\"text_item\", \"text\", \"a:first-of-type\"] ]` a:first-of-type - selector relative to the container\r\n\r\nNow in the html markup we will have only an array and two containers:\r\n\r\n```html\r\n\r\n <div data-menu=\"array\" >\r\n \r\n <div data-item=\"container\" >\r\n<a> text 1 </a>\r\n </div>\r\n \r\n <div data-item=\"container\" >\r\n<a> text 2 </a>\r\n </div>\r\n \r\n </div> \r\n\r\n```"
